# 端到端验收样例（MVP）

## 验收步骤
1. 启动后端与前端
2. 在“文档管理”上传一份石油工程教材 PDF，并等待状态变为 `completed`
3. 在“问答”选择教材范围（可不选，默认全部），提问下列问题
4. 验收要点：
   - 回答不胡编：无法定位时明确说明
   - 至少包含 1 条引用（或明确无法定位）
   - 引用包含页码与片段，能回到原文定位（后续可扩展为打开PDF页）

## 10 条问题集（示例）
- 解释“井眼稳定”的基本机理与主要影响因素。
- “钻井液密度”设计的一般思路是什么？有哪些约束条件？
- “地层压力”和“井底压力”的关系如何理解？给出教材中的定义出处。
- 欠平衡钻井的优点与风险分别是什么？尽量引用原文表述。
- 井漏的常见类型有哪些？各自的识别特征是什么？
- 钻头类型的主要分类方式是什么？不同类型适用场景如何？
- 套管设计需要考虑哪些工况载荷？给出教材对应章节页码。
- 解释“压裂裂缝导流能力”的含义及影响因素。
- 水泥浆性能评价指标有哪些？它们分别反映什么性质？
- 如果问题答案无法从教材片段得到，应如何回应（期望行为）？
