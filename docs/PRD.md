# 石油工程教材 RAG 问答 Web 应用（PRD Lite）

## 1. 背景与目标
面向石油工程教材（用户上传电子版）提供“可追溯引用”的问答能力：回答必须给出引用片段与页码，用户可一键打开原文定位。

### 成功标准（MVP）
- 可完成闭环：上传 PDF → 解析切分 → 建索引 → 问答 → 展示引用
- 可靠性：同一问题重复提问，引用页码稳定（允许回答措辞变化）
- 性能：问答 P95 < 12s（本地单机），上传解析异步执行
- 成本：记录每次问答的 token 与耗时；可配置预算上限（软限制）

## 2. 用户与场景
### 角色
- 学生：基于教材提问、查定义、对比概念、找步骤
- 教师：快速定位教材出处、组卷/出题辅助
- 管理员：维护文档、重建索引、删除数据

### 典型问题
- 概念解释：例如“井眼稳定的机理是什么？”
- 步骤/流程：例如“钻井液密度设计步骤？”
- 归纳对比：例如“欠平衡钻井与常规钻井对比”
- 引用定位：例如“教材里哪里定义了 X？给出页码”

## 3. 范围
### MVP（必须）
- 文档管理：上传 PDF、查看处理状态、删除、重建索引
- 解析与索引：按页保留元数据（页码、文档名、章节标题尽力抽取）、可配置切分策略
- 问答：对话式提问，返回回答 + 引用列表（文档名/页码/片段）
- LLM：至少接入 2 个通道（1 个云端、1 个本地 Ollama），UI 可切换

### 非 MVP（暂不做）
- OCR（扫描版 PDF、图片/公式识别）
- 多租户与复杂权限体系
- 复杂表格/公式的结构化抽取
- 在线协作与分享链接

## 4. 核心流程
### 4.1 上传与入库（异步）
1) 用户上传 PDF  
2) 服务端保存文件与元数据，创建 ingestion 任务记录（queued）  
3) 后台解析 PDF → 生成 page 文本与元数据  
4) 切分为 chunks，生成 embeddings  
5) 写入 PostgreSQL（pgvector）  
6) 任务状态更新为 completed / failed（失败记录错误摘要）

### 4.2 问答（同步）
1) 用户输入问题  
2) 检索 topK chunks（向量 + 可选关键词过滤）  
3) 将候选片段与问题喂给 LLM 生成回答  
4) 返回回答与引用（引用必须绑定 chunk/page 元数据）

## 5. 功能需求（验收口径）
### 文档管理
- 支持上传 PDF；单文件最大 200MB（可配置）
- 上传后可看到状态：queued / processing / completed / failed
- 支持删除文档（同时删除对应向量与 chunks）
- 支持重建索引（清空并重新解析切分）

### 问答
- 支持同一文档集内问答（可选择“全部文档”或“仅某文档”）
- 结果包含：
  - answer：自然语言回答
  - citations：至少 1 条（若检索置信不足，返回“无法在教材中定位”并 citations 为空）
  - 每条 citation 包含：doc_title、page_start/page_end、snippet、chunk_id

### 设置
- 可选择 LLM 通道与模型
- 可配置：temperature、topK、chunk_size/overlap（管理员）

## 6. 非功能需求
- 安全：不记录明文 API Key；日志脱敏；上传文件路径不可被直接枚举
- 可靠性：解析失败可重试；同一文档重复上传可识别并提示
- 可观测：记录 ingestion 耗时、embedding 耗时、检索耗时、LLM 耗时、token 统计

## 7. 数据与合规
- 提示用户仅上传拥有使用权的教材文件
- 支持删除文档及其派生数据（chunks、embeddings、对话记录可选）

## 8. 验收用例（MVP）
- 用 1 本教材 PDF 建库
- 准备 10 个问题（涵盖定义/步骤/对比/定位），验收：
  - 至少 7/10 能给出合理回答且引用页码一致
  - 3/10 若无法定位，必须明确说明并不胡编
